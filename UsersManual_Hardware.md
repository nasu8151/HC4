## 1. この文書について

本書は HC4 シリーズ（HC4, HC4E, HC8）のハードウェア構成を説明する文書です。
命令仕様やアセンブリ記法はソフトウェア編を参照してください。

## 2. 全体像

- 教育用途を主眼にしたシンプルな CPU
- スタックベース演算モデルを採用
- 74シリーズ汎用ロジックIC実装と Verilog 実装を想定
- 基本構成はハーバードアーキテクチャ

## 3. アーキテクチャ概要

### 3.1 データ幅

- 命令幅: 8 bit
- データバス幅: 4 bit（HC8は8bit）
- アドレスバス幅: 8 bit（HC4<sub>E</sub>は4bit、HC8は8bit）

モデル別の設計メモ:

| モデル             | 主な想定                  | 備考               |
| --------------- | --------------------- | ---------------- |
| HC4             | 3段スタック（A/B/C）+ 広いPC   | 標準モデル            |
| HC4<sub>E</sub> | 2段スタック（A/B）, PC 8 bit | 教育用軽量構成          |
| HC8             | 命令拡張あり                | LS, JL, LP などを使用 |

### 3.2 スタック構造

HC4 系列は演算オペランドとしてスタックレベルを使います。

- レベルA: 主オペランド / ロード先
- レベルB: 第2オペランド
- レベルC: 退避レベル（HC4E では制約あり）

ロード命令実行時は A <- 入力, B <- 旧A, C <- 旧B の順に押し込みます。

HC4E は有効スタック段数が 2 段（A/B）で、C段を使う前提の動作は制限されます。

### 3.3 クロックとリセット

- CLK 立ち上がりで命令実行の状態更新を行う構成を想定
- nRESET はアクティブLowの非同期リセットとして扱う実装が多い
- リセット解除直後のPC初期値は実装依存のため、シミュレーション時に明示確認する

## 4. 内部ブロック詳細

### 4.1 ALU

ALU は以下の演算を担当します。

- AND
- OR
- XOR
- 加算
- 減算

入力は主にスタック A/B、出力はデータバスへ接続されます。

命令とALUの対応（概略）:

| 命令   | 演算      | フラグ     |
| ---- | ------- | ------- |
| AD r | A + B   | C, Z 更新 |
| SU r | A - B   | C, Z 更新 |
| XR r | A XOR B | Z 更新    |
| OR r | A OR B  | Z 更新    |
| AN r | A AND B | Z 更新    |

注記:
- フラグ更新条件は実装差が出やすいため、Verilogと74系実装の両方で波形確認することを推奨

### 4.2 プログラムカウンタ（PC）

- 命令アドレスを保持
- 通常は命令実行ごとにインクリメント
- ジャンプ命令時は指定アドレスへ遷移

実装メモ:

- HC4 では広いPC幅（12bit）をとる
- HC4E では 8bit
- ジャンプ先はスタックCを最上位としてB, Aと連結するため、```LI #```命令でジャンプ先をロードしておいてください

### 4.3 命令デコーダ

命令デコーダは命令ビット列から制御信号を生成し、ALU、スタック、メモリ制御を切り替えます。
詳細な真理値に近い情報は以下を参照:

- extra/instruction.md

命令デコーダの役割:

- ALU演算種別の選択
- スタックのロード有効化
- メモリアクセス種別（読出し/書込み）選択
- ジャンプ条件評価とPC更新経路選択

## 5. バスとメモリ空間

### 5.1 バス

- データバス: 4 bit
- アドレスバス: 8 bit

### 5.2 メモリマップドI/O

HC4 系列はメモリマップドI/Oを想定します。
I/O デバイスはアドレスバスとデータバスへ接続してください。

実装時の目安:

- HC4 では 0xF0-0xFF をI/O領域として割り当てる構成を推奨（レファレンス実装はそうなっている）
- ただし最終的なI/O配置はアドレスデコーダ設計に依存

### 5.3 レジスタ領域

HC4シリーズではRAMのアドレス0x00~0x0Fを1命令でアクセスする命令があります。
この領域はプログラム作成時に前提となるため、RAM/I/Oの割り当て時に衝突しないように設計してください。

最低限のメモリマップ例:

| 範囲        | 用途       | 備考         |
| --------- | -------- | ---------- |
| 0x00-0x0F | レジスタ相当領域 | r0-r15 に対応 |
| 0x10-0xEF | RAM/作業領域 | 実装依存       |
| 0xF0-0xFF | I/O領域（例） | 実装で変更可     |

## 6. 制御信号

実装で最低限確認すべき信号:

- nRAM_RD: RAM 読み出し制御（アクティブLow）
- nRAM_WR: RAM 書き込み制御（アクティブLow）
- DATA_BUS[3:0]
- ADDRESS_BUS[7:0]
- CLK
- nRESET

内部制御線（ALU選択、スタック書き込み許可など）は実装方式により異なります。

## 7. モデル差分メモ

- HC4: 基本モデル
- HC4E: 2段スタック（A/B）かつ PC 8 bit の軽量系
- HC8: 命令拡張（例: LS, JL, LP）を持つ

命令差分の一覧はソフトウェア編を参照してください。

差分の要点:

| 項目      | HC4               | HC4E         | HC8       |
| ------- | ----------------- | ------------ | --------- |
| スタック段数  | 3段（A/B/C）         | 2段（A/B）      | 3段（A/B/C） |
| PC幅     | 実装により広幅（例:12 bit） | 8 bit        | 実装依存      |
| 命令セット   | 基本                | 制限あり         | 拡張あり      |
| I/Oレジスタ | 実装依存              | r14, r15 を使用 | 実装依存      |

注記:
- HC4E のアドレス空間は 16ニブル（r0-r15）として扱う運用を基本とする
