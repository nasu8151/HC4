# 表の読み方

HC4シリーズの命令幅は 8 bit です。

- `r`: 4 bit レジスタ番号（`r0` から `r15`）
- `#i`: 4 bit 即値
- `[AB]`: スタック A/B を連結した間接アドレス（MSB は B）
- `[ABC]`: スタック A/B/C を連結した間接アドレス（MSB は C）
- `PC`: Program Counter

予約済み（Reserved）命令は使用しないでください。

# 命令テーブル

## HC4

| bit 5-4→<br>7-6↓ | 00         | 01         | 10                      | 11         |
| ---------------- | ---------- | ---------- | ----------------------- | ---------- |
| 00               | `SM`       | `SC r`     | `SU r`                  | `AD r`     |
| 01               | `XR r`     | `OR r`     | `AN r`                  | `SA r`     |
| 10               | `LM`       | `LD r`     | `LI #`                  | (Reserved) |
| 11               | (Reserved) | (Reserved) | `JP flag`, `NP`         | (Reserved) |

| 01               | `XR r`     | (Reserved) | (Reserved)              | `SA r`     |
| 10               | (Reserved) | `LD r`     | `LI #`                  | (Reserved) |
| 11               | (Reserved) | (Reserved) | `JP flag`, `NP`         | (Reserved) |

# 命令一覧

## 論理演算・算術演算

| 命令   | Opc  | Opr | フラグ変化 | 動作       | 説明                   |
| ------ | ---- | --- | ---------- | ---------- | ---------------------- |
| `SU r` | 0010 | r   | C, Z       | r <= A-B   | 減算結果を r に格納    |
| `AD r` | 0011 | r   | C, Z       | r <= A+B   | 加算結果を r に格納    |
| `XR r` | 0100 | r   | Z          | r <= A^B   | XOR 結果を r に格納    |
| `OR r` | 0101 | r   | Z          | r <= A\|B  | OR 結果を r に格納     |
| `AN r` | 0110 | r   | Z          | r <= A&B   | AND 結果を r に格納    |

## レジスタ・メモリアクセス

| 命令     | Opc  | Opr  | フラグ変化 | 説明                               |
| -------- | ---- | ---- | ---------- | ---------------------------------- |
| `SM`     | 0000 | 0000 | Z          | C を `[AB]` に格納                 |
| `SC r`   | 0001 | r    | Z          | C を r に格納                      |
| `SA r`   | 0111 | r    | Z          | A を r に格納                      |
| `LM`     | 1000 | 0000 |            | `[AB]` から A にロード             |
| `LD r`   | 1001 | r    |            | r から A にロード                  |
| `LI #i`  | 1010 | i    |            | 即値を A にロード                  |
| `LS #i`* | 1100 | i    |            | 即値ロード + A シフト（HC8のみ）   |

注記:
- `LS #i` は HC8 のみ。

## システム制御

| 命令    | Opc  | Opr  | 説明                           |
| ------- | ---- | ---- | ------------------------------ |
| `JP`    | 1110 | 0000 | `[ABC]` へジャンプ             |
| `JP C`  | 1110 | 0010 | C フラグが 1 のときジャンプ    |
| `JP NC` | 1110 | 0011 | C フラグが 0 のときジャンプ    |
| `JP Z`  | 1110 | 0100 | Z フラグが 1 のときジャンプ    |
| `JP NZ` | 1110 | 0101 | Z フラグが 0 のときジャンプ    |
| `NP`    | 1110 | 0001 | 何もしない（NOP 相当）         |

# 命令解説

## 論理演算・算術演算

これらの命令は主にスタック A と B を入力に取り、演算結果を指定レジスタ r に格納します。

## レジスタ・メモリアクセス

- Store 系命令は A または C をレジスタ/メモリへ保存します。
  - `SA r`: A を参照
  - `SC r`, `SM`: C を参照
- Load 系命令は A に値を読み込みます。
  - ロード時は A/B/C が 1 段ずつ押し下がります（A <- new, B <- old A, C <- old B）。
  - `LD r`: レジスタ参照
  - `LI #i`: 即値参照
  - `LM`: `[AB]` で指定されたメモリアドレス参照
- `LS #i` と `LI #i` では 2進リテラル（例: `0b1010`）と 16進リテラル（例: `0xA`）を使用できます。

## システム制御

- `NP` は何もしません。
- `JP` 系命令は条件に応じて PC を更新します。

## HC4E の注意

- 有効なスタックレベルは A/B が中心で、基本モデルより制約があります。
- アドレス空間は 16 nibble を基本とする構成です。
- レジスタアドレッシング（`r`）主体で使用します。
- I/O レジスタは `r14`, `r15` に配置します。

# 命令ごとの使用例と注意点

ここでは「最小限の命令列で書ける」ことを重視して、短い例を示します。

## AD r / SU r

例:

```asm
ld #0x3
ld #0x5
ad r1
```

注意点:

- 演算元は主にスタック A/B で、結果は指定レジスタ r に書き込まれます。
- C, Z フラグが更新されるため、直後の条件分岐に影響します。

## XR r / OR r / AN r

例:

```asm
ld #0xA
ld #0x3
xr r2
```

注意点:

- Z フラグ更新を利用して、ゼロ判定分岐を書けます。
- C フラグは算術系ほど明確に使わないため、分岐条件を混同しないようにします。

## SA r / SC r / SM

例:

```asm
ld #0x9
sa r0
```

注意点:

- `SA r` は A を保存、`SC r` と `SM` は C を保存します。
- アセンブラ実装によっては `[AB]` への書き込み表記が `SM` または `SC [AB]` として扱われるため、使用アセンブラの仕様を確認してください。

## LD r / LI #i / LM

例:

```asm
li #0xF
ld r1
lm
```

注意点:

- ロード時はスタックが押し下がるため、意図しない値が B/C に残ることがあります。
- `LM` は `[AB]` の内容を使うため、事前に A/B を正しく準備してください。

## JP 系 / NP

例:

```asm
ld #0
ld #0
ld #0x4
jp
```

注意点:

- `JP` の飛び先はスタック内容で決まるため、ロード順序を逆にすると誤ジャンプします。
- `NP` は遅延挿入や波形観測時の区切りに使えます。

# チュートリアル: LED点滅（lchika.asm）

対象ファイル: `extra/lchika.asm`

```asm
np
ld #0xF
ld #0xF
ld #0
sc [AB]
ld #0
ld #0xF
ld #0
sc [AB]
ld #0
ld #0
ld #0x1
jp
```

流れ:

1. `ld #0xF`, `ld #0xF`, `ld #0` で書き込み先アドレスと値をスタックに準備
2. `sc [AB]` でメモリマップドI/Oへ出力値を書き込む
3. 別の値を書き込んで LED ON/OFF を切り替える
4. 最後に `jp` で先頭に戻り、点滅を繰り返す

このサンプルは「ロード」「ストア」「ジャンプ」の最小構成で、最初に動かす確認用として使えます。

行単位の解説:

1. `np`: 初期NOP
2. `ld #0xF`: アドレス上位側の準備
3. `ld #0xF`: アドレス下位側の準備
4. `ld #0`: 出力データ（例: OFF）
5. `sc [AB]`: I/Oへ書き込み
6. `ld #0`: 次の遷移用データ準備
7. `ld #0xF`: 同一I/O領域指定
8. `ld #0`: 出力データ再設定
9. `sc [AB]`: I/Oへ再書き込み
10. `ld #0`: ジャンプ先下位準備
11. `ld #0`: ジャンプ先中位準備
12. `ld #0x1`: ジャンプ先上位準備
13. `jp`: 先頭側へ戻る

注記:

- `sc [AB]` は利用アセンブラによって `sm` 相当として解釈される場合があります。
- 実機で点滅しない場合は、I/Oデコード範囲と極性（アクティブHigh/Low）を最初に確認してください。

# トラブルシューティング

- 予約済み命令を使っていないか確認する
- I/O アドレスの割り当てが回路と一致しているか確認する
- `JP` の飛び先アドレス（スタック内容）が意図どおりか確認する
- フラグ条件ジャンプを使う場合、直前命令の C/Z 更新を確認する
- `LD`/`LI` 後のスタック押し下がりで、想定外の値が B/C に残っていないか確認する
- HC4E では 2段スタック前提のため、C段前提のコードを書いていないか確認する
- アセンブラ方言差（`SM` と `SC [AB]`）でエンコード結果が変わっていないか確認する
